# COMP3011 Search Engine

A command-line search engine that crawls [quotes.toscrape.com](https://quotes.toscrape.com/), builds an inverted index with TF-IDF scoring, and lets you search for pages containing one or more keywords.

---

## Architecture Overview

```
search_engine/
├── src/
│   ├── crawler.py   – BFS web crawler with politeness window
│   ├── indexer.py   – Inverted index builder with TF-IDF scoring
│   ├── search.py    – JSON persistence and high-level search interface
│   └── main.py      – Interactive command-line shell
├── tests/
│   ├── test_crawler.py
│   ├── test_indexer.py
│   └── test_search.py
├── data/
│   └── index.json   – (generated by `build` command)
├── requirements.txt
└── README.md
```

### Design Decisions

**Crawler** uses Breadth-First Search (BFS) to crawl all pages reachable from the base URL. BFS was chosen over DFS because it discovers all pages at a given depth before going deeper, making progress more uniform. The crawler normalises URLs (strips trailing slashes and fragments) to avoid revisiting the same page with slightly different URLs. All failed HTTP requests are handled gracefully – the crawler logs the error and continues.

**Inverted Index** maps each unique word to a postings list containing: term frequency, word positions (0-based), and a pre-computed TF-IDF score. Storing positions enables future work on phrase matching. The index is a plain Python dict, which makes it JSON-serialisable without third-party libraries.

**TF-IDF Scoring** — `TF = term_freq / doc_length` (normalised term frequency); `IDF = log(N / doc_freq)` (inverse document frequency). This rewards words that are frequent in a page but rare across the collection, giving more relevant results than raw term counts.

**Storage** uses JSON rather than pickle because JSON is human-readable (the index file can be inspected in any text editor), portable across Python versions, and safe to load (no arbitrary code execution risk).

**Multi-word queries** use intersection semantics: a page must contain *all* query words to appear in results. Pages are ranked by the sum of their TF-IDF scores across all query terms.

## Installation

### Prerequisites

- Python 3.10 or later
- pip

### Steps

```bash
# 1. Clone the repository
git clone <your-repo-url>
cd search_engine

# 2. (Optional) create a virtual environment

pip install -r requirements.txt
```

## Usage

Start the interactive shell from the `src/` directory:

```bash
cd src
python main.py
```

You will see the welcome banner and a `>` prompt.

### Commands

#### `build`

Crawls the target website, builds the inverted index, and saves it to `data/index.json`.

```
> build
Starting crawl of https://quotes.toscrape.com/
Politeness window: 6.0s between requests
...
Index saved to 'data/index.json' (245.3 KB, 1,842 terms, 11 documents).
```

**Note:** The build takes roughly 1–2 minutes due to the mandatory 6-second politeness window.

#### `load`

Loads a previously saved index from `data/index.json`. Must be run before `print` or `find` if you have not just run `build`.

```
> load
Index loaded from 'data/index.json' (245.3 KB, 1,842 terms, 11 documents).
```

#### `print <word>`

Prints the full inverted index entry for a word, including document frequency, term frequency, TF-IDF score, and word positions.

```
> print nonsense
Inverted index for 'nonsense':
  Document frequency: 2

  URL:       https://quotes.toscrape.com/
  Term freq: 1
  TF-IDF:    0.016234
  Positions: [42]

  URL:       https://quotes.toscrape.com/page/2/
  Term freq: 2
  TF-IDF:    0.032468
  Positions: [17, 89]
```

The search is **case-insensitive**: `print Nonsense` returns the same result.

#### `find <query>`

Finds all pages containing every word in the query. Results are ranked by TF-IDF score.

```
> find indifference
Found 1 page(s) for query 'indifference':

  1. https://quotes.toscrape.com/page/3/
     TF-IDF score: 0.024511
```

```
> find good friends
Found 2 page(s) for query 'good friends':

  1. https://quotes.toscrape.com/page/1/
     TF-IDF score: 0.031200
  2. https://quotes.toscrape.com/page/5/
     TF-IDF score: 0.019800
```

#### `help`

Prints the command reference.

#### `quit` / `exit`

Exits the shell.

## Testing

Run the full test suite from the project root:

```bash
pytest tests/ -v
```

Run with coverage report:

```bash
pytest tests/ -v --cov=src --cov-report=term-missing
```

### Test Strategy

Tests are organised into three files mirroring the source modules:

- **test_crawler.py** — URL normalisation, link extraction, HTTP error handling, BFS traversal (all HTTP calls are mocked; no real network requests are made in tests).
- **test_indexer.py** — Tokenisation, HTML text extraction, index building, TF-IDF correctness, multi-word intersection search, edge cases (empty queries, unknown words, case sensitivity).
- **test_search.py** — JSON save/load round-trip integrity, command output formatting, error messages for unloaded index, corrupt file handling.

## Politeness

The crawler waits **6 seconds** between requests (configurable via `POLITENESS_WINDOW` in `main.py`). This complies with the assessment requirement and avoids overloading the target server.

---

## Logging

The application writes structured logs to `search_engine.log` in the project root. The log includes timestamps, crawled URLs, and any errors encountered during crawling or index loading.
